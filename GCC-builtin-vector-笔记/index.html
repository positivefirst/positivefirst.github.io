GCC builtin vector 笔记

发表于 2022-04-05

GCC builtin vector很像固定长度的valarray，它存放了若干个相同类型的变量，并支持对所有变量统一施加运算。在指令集的加持下，它的性能有巨大的提升，有时甚至可以“暴力碾标算”。

下文的vector均指GCC builtin vector。

这里是vector的官方文档，介绍了它的使用方法。下面一行代码定义了u256类型，长度为32字节，存放8个unsigned类型。

使用vector要在代码开头加#pragma GCC target("avx,avx2")，如果没有这句话，vector会跑得比for循环+普通数组还要慢。洛谷IDE有编译选项-march=native，这句话就可以不写。此外以下代码测试时都有编译选项-O2 -std=c++11。

还有一种指令集的写法需要#include<immintrin.h>，各种操作形如_mm256_xxxxxxxxxxx，速度和vector差不多，但写起来不如vector方便。

如果没有写#pragma GCC target("avx,avx2")，vector用时达到了丢人的17686ms。不过这句话对暴力和循环展开的速度几乎没有影响。在接下来的试验中，这句话对不使用vector的代码也几乎没有影响。

第二行的7780ms是在数组大小为1024时测的，如果数组大小改为1025，用时降为2187ms。这大概证实了“不要开二的幂次大小的数组”的传说（知乎问题）。不过调整数组大小对其他方法的速度几乎没有影响。

bitset
