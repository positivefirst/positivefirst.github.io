GCC builtin vector 笔记

发表于 2022-04-05

GCC builtin vector 很像固定长度的 valarray，它存放了若干个相同类型的变量，并支持对所有变量统一施加运算。在指令集的加持下，它的性能有巨大的提升，有时甚至可以“暴力碾标算”。

下文的 vector 均指 GCC builtin vector。

这里是 vector 的官方文档，介绍了它的使用方法。下面一行代码定义了 u256 类型，长度为 $32$ 字节，存放 $8$ 个 unsigned 类型。

使用 vector 要在代码开头加 #pragma GCC target("avx2")，如果没有这句话，vector 会跑得比 for 循环+普通数组还要慢。洛谷 IDE 有编译选项 -march=native，这句话就可以不写。此外以下代码测试时都有编译选项 -O2 -std=c++11。

区间加区间求和

如果没有写 #pragma GCC target("avx2")，vector 用时达到了丢人的 $17686\texttt{ms}$。不过这句话对暴力和循环展开的速度几乎没有影响。在接下来的试验中，这句话对不使用 vector 的代码也几乎没有影响。

第二行的 $7780\texttt{ms}$ 是在数组大小为 $1024$ 时测的，如果数组大小改为 $1025$，用时降为 $2187\texttt{ms}$。这大概证实了“不要开二的幂次大小的数组”的传说（知乎问题）。不过调整数组大小对其他方法的速度几乎没有影响。
