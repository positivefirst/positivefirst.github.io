randmd5

发表于 2021-06-18

基于MD5的随机数生成器

概述

randmd5使用字符串作为种子，生成随机数，也可以作为随机数引擎。

它可以在C++98下通过编译（比rand强多了），也可以和C++11的随机数分布配合使用。

MD5运算的代码来自百度百科，有删改。

使用方法

复制本文最后的代码，保存为randmd5.h（和使用这个头文件的代码保存在同一个文件夹中）。

首先要#include"randmd5.h"。声明的随机数变量类型就是randmd5。

后文中默认typedef unsigned int ui;。

构造函数

s是随机数种子。如果s的长度超过4000，超过部分会被忽略。

a,b,c,d是MD5运算要用到的初始值，如果不指定，默认为a=0x12345670u,b=0x89abcdefu,c=0x20210118u,d=0xccfccfacu。

随机数引擎

为了适应C++11的随机数分布（如uniform_int_distribution和uniform_real_distribution），一般不会用到。

min()的返回值为0，max的返回值为0xffffffffu。

生成随机数

len是希望生成的随机数的二进制长度，也就是返回一个均匀分布的在$0 \sim 2^{len}-1$范围内的随机数。

如果len<1||len>32，返回0。

更改种子

s是新的随机数种子。如果s的长度超过4000，超过部分会被忽略。

实际效率

准备

使用各自的默认种子，生成一千万个随机数（$0$或$1$）。

结果

随机数生成方式 u(e) r(1) u(r)
$1$的个数（以$5$百万为基准） $-458$ $+251$ $-2018$
用时（秒） $2.656$ $3.859$ $62.728$

结论

三种生成方式产生的随机数质量差不多，$1$的个数均属于正常范围。

相比之下，u(r)的速度很慢，但如果不要求生成那么多随机数，还是可以接受的。
