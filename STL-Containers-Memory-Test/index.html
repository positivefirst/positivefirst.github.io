STL 容器空间常数测试

发表于 2021-08-26

STL 容器的空间复杂度基本都是线性，但实际占用内存可以相差十多倍。

测试环境：洛谷 IDE，C++11，不开 O2。

使用各容器插入 $10^6$ 个 int 类型。理论上占用约 4000kb，当然程序其他部分也会消耗内存。

容器 内存/kb sizeof 结果
C风格数组 4492 4000000
array 4472 4000000
vector1 5988 24
vector2 4468 24
queue 4728 80
priority_queue 5988 32
deque 4660 80
stack 4600 80
list 31752 24
set 47472 48
unordered_set 50292(39968) 56

vector1 是创建空 vector，用 push_back 加入元素。vector2 是创建长度为 $10^6$ 的 vector，用 operator[] 修改元素。priority_queue 的测试结果波动很大，最小 4906kb，最大 6500kb。

（更新于 2023/12/9）发现 GCC 偷偷优化了 unordered_set，括号内的数值是用 C++14（GCC 9）测出的。可惜当年没记录编译器版本。

考场挂分技巧：开很大的 deque/queue/stack 数组。每个 deque 在创建时都会提前申请不少内存以容纳将来可能会加入的元素（详见 cppreference），容易使人低估实际占用内存。

仍在洛谷 IDE，上面一段代码输出 7812，但内存占用 67684kb。更多次的测试显示平均每个空的deque 占用约 689 字节。
