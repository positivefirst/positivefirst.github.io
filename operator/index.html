题目背景

选自https://www.luogu.com.cn/problem/T206725，有删改。

题目描述

你有 $4$ 个数，初值都是 $64$ ，每次你要在给出的操作中二选一，使这 $4$ 个数均位于 $[0,127]$ 中尽可能久。
技能 $2$ ：所有 $+$ 操作变成 $-$ 操作， $-$ 操作变成 $+$ 操作，使用两次技能 $2$ 效果会消失。
注意：交互库输出操作时会自动转换。
技能 $3$ ：跳过一次选择。
技能 $4$ ：你需要额外输入两个属于 $[0,3]$ 的整数，效果是交换给定下标的两个数。
使用技能 $4$ 或输入错误后，还要输入一个属于 $[0,1]$ 的整数（在给出的操作中二选一）。

输入格式

每次你需要输入一个属于 ${0,1,3,4}$ 的数。输入 $0$ 或 $1$ 表示选择执行的操作编号，输入 $3$ 或 $4$ 表示使用对应技能。

输出格式

交互库每局会在第一行给出当前是第几局。第二行给出空格隔开的 $4$ 个整数，代表目前这 $4$ 个数的值。

第三、四行给出操作。每个操作包含 $5$ 个用空格隔开的字符串。前四项代表对应数将发生的变化（可能是 +-^|&!# ，前 $5$ 个后跟操作数，同C++语法； ! 表示该数不变；# 表示该数变为 $64$ ），最后 $1$ 项代表执行该操作引发的技能编号（ ! 表示无技能）。

第五行是技能的状态，空格隔开。第一项为 *1 （不使用技能 $2$ ）或 *-1（使用技能 $2$ ），第二项为技能 $3$ 可用次数，第三项为技能 $4$ 可用次数。

结束后输出*，后跟一个数，表示游戏的局数。

opt.txt的格式：

第一行一个正整数 $n(0\leq n \leq 50)$ 。

接下来 $2n$ 行，每行一个操作（ $5$ 个用空格隔开的字符串）。其中第 $2i-1$ 个操作和第 $2i$ 个操作为一对操作。

示例：

注意：如果交互库所在文件夹没有opt.txt，交互库会生成默认的opt.txt 。 为获得更（du）好（liu）的游戏体验，请删除旧版本附带的opt.txt。

本题中Error的类型:
Error 1 : opt.txt中 $n$ 超出范围$[0,50]$ .
Error 2 : opt.txt中+-后的操作数超出范围 $[0,10]$ .
Error 3 : opt.txt中^|&后的操作数超出范围 $[0,127]$ .
Error 4 : 读入每次的操作数$y \notin {0,1,3,4}$.
Error 5 : 在技能 $3$ 和技能 $4$ 剩余次数为 $0$ 的情况下使用了技能.
Error 6 : 在技能 $4$ 中输入的下标不在$[0,3]$范围内.
